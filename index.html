4. PackML state diagram

4.1 Definitions and key concepts

[0010] PackML:

[0010] General comments on the state machine. What does the box around states Idle..Completed mean? What does the box around all states but Aborting and Aborted mean?

[0010] PackML states are meant to convey information about the general status of the machine. It is common for any given state to have multiple substates...

[0010] Clarify difference between Hold and Suspend states, including autorestart for Suspend state.

[0010] Clarify difference between Stopped and Aborted states.

The PackML state diagram is meant to represent in a broad way the behaviour of a generic machine. It is meant to be more descriptive than proscriptive and, as such, should not be viewed as a programatic trick. States have specific meanings. For example, if machine processing has paused due to an upstream blockage, then the machine is in Suspended state, regardless of internal PLC logic (are any other messages to the contrary).

Processing on a given machine has paused due to a low level fault on a glue tank. From a PackML perspective, what state is the machine in?

A machine has processed all units required for a given batch and has come to rest. From a PackML perspective, what state is the machine in?

A machine has come to rest due to an emergency stop. From a PackML perspective, what state is the machine in?

Give an example of a machine which might not implement the Suspended state in Production mode.
